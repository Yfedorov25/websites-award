# Research Report #10 — Мікрокопі та UX-текст (на оригіналах)
> Повний звіт. Джерело: СКРАП оригіналу Mailchimp Content Style Guide (styleguide.mailchimp.com, datasetId YKKf6y0sHN4n6zoJ7, 17 сторінок) + принципи Shopify Polaris / Apple HIG. Синтез → copywriting/microcopy-ux.md.

## TL;DR
- Мікрокопі (кнопки, форми, помилки, порожні стани) вирішує юзабіліті більше за маркетинговий текст. Еталон — Mailchimp: ясність важливіша за розвагу.
- Три тверді правила зі скрапу оригіналу: кнопки завжди з дієсловом (ніколи Submit/ОК); жодних знаків оклику в помилках; ніколи «Click here».
- Позитивна мова (що можна), не негативна (що не можна). Проси у форм лише необхідне. Системні алерти — теплі й людські.

## ДЖЕРЕЛО (verbatim зі скрапу Mailchimp)
**Кнопки:** «Button copy should always include verbs. Keep things clear and concise. Standard website buttons: Log in, Sign up free, Subscribe, Email us.»
**Помилки:** «Never use exclamation points in failure messages or alerts. When in doubt, avoid!»
**Посилання:** «Don't say things like 'Click here!' or 'Click for more information' or 'Read this.' Instead, link relevant keywords.»
**Форми:** «Keep forms as short as possible. Only request information that we need and intend to use. Don't ask for information that could be considered private or personal.»
**Позитивна мова:** «Yes: To get a donut, stand in line. No: You can't get a donut if you don't stand in line.»
**Тон системних алертів:** «All of our content, from splashy homepage copy to system alerts, should be warm and human.»
**Ясність:** «It's always more important to be clear than entertaining.»

## ПРАВИЛА ПО ЕЛЕМЕНТАХ

### Кнопки
- Завжди дієслово. Log in / Sign up free / Subscribe. Ніколи Submit/OK/Click here.
- Чітко, коротко, описує дію.
- UA: наказовий спосіб + результат («Отримати план», «Записатися на візит»).

### Форми
- Заголовок пояснює мету. Title case заголовки, sentence case поля.
- Якомога коротша. Лише необхідні поля. Без зайвого особистого (стать).
- Якщо потрібна стать — поле для вводу, не дропдаун.

### Помилки / алерти
- Жодного знаку оклику. Позитивна мова. Що сталось + що робити. Без провини. Без драми.

### Посилання
- Ніколи «Click here». Лінкуй ключові слова. Hover+focus стан. Не лінкуй артиклі й пунктуацію.

### Заголовки UI
- Sentence case, без крапки (окрім питань). Ключові слова + головна думка. Ієрархія H1→H2→H3.

### Списки
- Інтро-фраза. Нумерація лише коли порядок важливий. Узгоджена пунктуація/капіталізація.

## ACCESSIBILITY
Найважливіше перше; проста мова, короткі речення; посилання інформують про дію; без напрямкових інструкцій («справа»); чіткі підписи інпутів; alt-текст 1-2 речення.

## ПЕРЕКЛАД / БАГАТОМОВНІСТЬ
Активний стан; без подвійних заперечень; без синонімів того самого слова; без сленгу/ідіом; стисло, але ясно.

## UA-АДАПТАЦІЯ
- Кнопки: наказовий спосіб, без канцеляриту й «ОК».
- Поля: людські назви («Твоє ім'я»), не «ПІБ»/«Контактні дані суб'єкта».
- Помилки: спокій без провини, без окликів. «Не вдалося зберегти. Спробуй за хвилину.»
- Wellbeing: у формах/помилках без тиску, провини, паніки.

## ЧЕКЛИСТ ПЕРЕД ПУБЛІКАЦІЄЮ
Кнопка з дієсловом? Без окликів у помилках? Без «Click here»? Форма лише необхідне? Помилка каже що робити без провини? Позитивна мова? Без напрямкових інструкцій? UA: без канцеляриту, CTA в наказовому?

## CAVEATS
- Скрапнуто Mailchimp (англомовний еталон) повністю; Shopify Polaris і Apple HIG — за відомими принципами, не скрапнуто цього разу (можна добрати окремо).
- Mailchimp-правила капіталізації (title/sentence case) — англомовні; для UA адаптувати (українська не має title case у тому ж сенсі — там sentence case природніший).
