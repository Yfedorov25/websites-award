# 🏗️ ПАТЕРН — INDUSTRIAL / DEEPTECH / промисловий продукт (B2B технічний)
> З розбору Vaulk. Тип: утилітарний-технічний. ЦА: B2B, держ, інфраструктура, інженери. Застосовно: Phoenix-tec (зерносушарки), виробництво, агротех, промислове обладнання.

## КАРТА СЕКЦІЙ:
1. ПРЕЛОАДЕР — лічильник (опц., для солідності).
2. HERO — ПРОДУКТОВА СПЕЦИФІКАЦІЯ + ключова вигода + строк («hardened shelters, deployed in 7 days»). Що+перевага+швидкість. → R_hero (manifesto-варіант, стриманий)
3. CORE CONCEPT — 2-3 стовпи суті продукту (Armored / Modular).
4. SYSTEM CAPABILITIES — технічні підсистеми/компоненти (що всередині).
5. ТЕХНОЛОГІЯ/МАТЕРІАЛ — характеристики як докази (термін служби, міцність, стандарти).
6. ПРОЦЕС/ЛОГІСТИКА — кроки 001-003 з часовою шкалою (виробництво→доставка→монтаж).
7. МАТРИЦЯ — охоплення (типи застосування/загроз/конфігурацій).
8. USE CASES — нумеровані сценарії (001-005: різні ЦА/застосування).
9. FAQ — детальні технічні відповіді (відмінності, час, стандарти, партнери).
10. КОНТАКТ — форма (B2B/приватний поділ). → R_cta + форма
## НОРМИ: 9-11 секцій, 15-20 екранів (насичено технікою — B2B читають специфіки). H1 стриманий (не гігантський крик).
## ПАЛІТРА: індустріальна/стримана (графіт, метал, земляна, темна). Технічна, не яскрава.
## ТИПО: display + МОНО-шрифт (Chivo Mono/IBM Plex — «інженерна точність, дані»). Нумерація скрізь (001/002).
## КОПІ-РЕГІСТР: технічна КОНКРЕТИКА+цифри+стандарти+сертифікації як довіра. Big Idea=специфікація+вигода+строк. Без емоцій — факти. Нумерація=системність.
## ЕМОЦІЯ: довіра через інженерну точність, стандарти, конкретні цифри. Надійність.
## ВІДТВОРЮВАНІСТЬ: 🟢-🟡. Vue/Next+GSAP+Lenis. WebGL дозований (замінний 3D-рендерами продукту/відео).
## ДЛЯ PHOENIX-TEC: зерносушарки = той самий патерн (специфікація+характеристики+процес+use cases агросектору+довіра через цифри/роки роботи).
