# 🏗️ ПАТЕРН — SaaS / продукт лендинг (утилітарний, конверсійний)
> Тип: утилітарний. Темп швидкий 0.2-0.4s. Воронка-фокус, докази, CTA.

## КАРТА СЕКЦІЙ:
1. HERO — продукт + 1 вигода (3-6 слів) + CTA «Спробувати/Demo» + скрін/відео продукту. → R_hero
2. ЛОГО клієнтів (trust-bar) — marquee лого. Довіра одразу.
3. ПРОБЛЕМА→РІШЕННЯ — біль ЦА, тоді як продукт вирішує.
4. ФІЧІ (3-6) — сітка з іконками, stagger. → R_services (варіант сітка)
5. ЯК ПРАЦЮЄ — 3 кроки (часто pinned-scroll E3).
6. ДОКАЗИ — цифри/відгуки/кейси. → R_about (counter)
7. ЦІНИ — пакети-колонки (продуктизація). → R_services (варіант C)
8. FAQ — акордеон.
9. CTA-фінал — «Почни безкоштовно». → R_cta
10. FOOTER. → R_footer
## НОРМИ: 8-10 секцій, 12-16 екранів. Швидкий рух. Світла або deep-tech база. Багато доказів.
## ЕМОЦІЯ: ясність→довіра→бажання спробувати. Schwartz: проблема для холодної ЦА вгорі.

## + ЗБАГАЧЕННЯ (з Elva — AI-продукт):
- ОЛЮДНЕННЯ AI: «personal AI director/agent», не «алгоритм/нейромережа». Персона-помічник.
- ПРОБЛЕМА через ЕМОЦІЮ/біль ЦА, не через відсутню функцію («every video holds a feeling, but it gets lost»).
- Big Idea = РЕЗУЛЬТАТ-ПОЧУТТЯ, не технологія («everyday videos into emotional stories»).
- «Як працює» = прості кроки дієсловами (upload→tell→edits→pick).
- Емоційний регістр для tech-продукту = диференціація від сухих AI-конкурентів.

## + ЗБАГАЧЕННЯ (з aqryl — fintech/dashboard SaaS):
- DASHBOARD-ПОКАЗ (не phone): великі скріни/відео робочого простору, кожна фіча = свій екран продукту.
- ФІЧА ЧЕРЕЗ СЦЕНАРІЙ-вигоду, не назву: «Journal that still reads clean months later» (як допоможе), не «журнал угод».
- АНТИТЕЗА-контраст з конкурентами: «sessions, not noise», «visible, not loud» (виділення підходу).
- Mono-шрифт (Geist Mono) для метрик/цифр = fintech-точність.
- Big Idea через БІЛЬ ЦА: «deserve more than guessing» (емоція невпевненості).
