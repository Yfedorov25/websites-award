# 📘 ФОРМАТ ПЛЕЙБУКА (майстер-клас побудови аспекту)
> Плейбук ≠ рецепт. Рецепт = специфікація (числа). Плейбук = ПОКРОКОВА методика побудови з варіантами, рішеннями, помилками.
> Claude у META читає плейбук відповідного аспекту й будує крок-за-кроком, обираючи варіант під контекст.

## СТРУКТУРА КОЖНОГО ПЛЕЙБУКА:
1. **ЩО ЦЕ + РОЛЬ** — навіщо аспект, яку роботу виконує.
2. **ТИПИ/ВАРІАНТИ** — 3-6 варіантів виконання + КОЛИ кожен (під ціль/нішу).
3. **ПОКРОКОВА ПОБУДОВА** — нумеровані кроки від нуля до готового (рішення на кожному).
4. **ДЕРЕВО РІШЕНЬ** — якщо X → роби Y (вибір варіанту за контекстом).
5. **ТОЧНІ ПАРАМЕТРИ** — числа з рецептів/норм (clamp, easing, stagger).
6. **КОД-СКЕЛЕТИ** — готові фрагменти для кожного варіанту.
7. **ТИПОВІ ПОМИЛКИ ДЖУНІОРА** — чого НЕ робити (з A1 червоних прапорців).
8. **РЕАЛЬНІ ПРИКЛАДИ** — як це зроблено на еталонах (з D-розборів).
9. **МОБІЛЬНИЙ** — перебудова.
10. **ЧЕК** — критерії A1 для цього аспекту (виконано/ні).
