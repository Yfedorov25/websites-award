## A0. АНАТОМІЯ РУХУ ЕЛЕМЕНТІВ (entrance/exit · швидкість · easing · stagger · зв'язність) ★★★
> Найважливіше для емоції: НЕ що рухається, а ЯК входить/виходить, з якою швидкістю, кривою, і як елементи пов'язані. Це фіксуємо на кожному сайті.

### ⭐ ПІДСУМОК MOTION-БЛОКУ (20 студій) — головні висновки про рух:
1. **Easing: індустрія тяжіє до м'яких easeOut** (Quad/Cubic). Дефолт = `cubic-bezier(0.25,0.46,0.45,0.94)` easeOutQuad (4× підтверджень) + `cubic-bezier(0.215,0.61,0.355,1)` easeOutCubic. Для драми — expo (Cuberto). Для performance — Material+linear (basement).
2. **Тривалість визначає клас емоції:** 0.06-0.2s мікро-відгук / 0.3-0.6s UI+японський темп / 1-1.25s драматичні преміальні появи. «Спектакль»-студії = 1-1.25s, performance = 0.06-0.3s.
3. **Класи руху елементів:**
   - ПОЯВА: opacity+transform (fade-up) — найуніверсальніший (всі).
   - РОЗКРИТТЯ: clip-path+opacity (шторка/завіса) — театральне (Tendril).
   - МОРФІНГ ФОРМИ: max-width/max-height/border-radius — трансформація на місці (Adoratorio).
   - СЦЕНА: scroll-driven камера в WebGL (Lusion/AT).
4. **Stagger = зв'язність:** ланцюг delay (крок 0.05-0.2s) + одна easing-крива на весь сайт = елементи як ланки хвилі (14islands +0.15s). Тригер scroll-into-view.
5. **Темп = декларація цінностей** (рух «говорить» позиціонування ще до тексту).
6. **WebGL не обов'язковий для award** (Cuberto/Use All Five — топ без 3D, чистий GSAP+CSS). Емоція з точного руху, не з 3D.

### Виміри, які трекаємо:
- **Entrance** — звідки приходить: translateY (знизу/зверху), opacity (проявлення), scale (наближення/віддалення), clip-path/mask (розкриття-шторкою), rotate/skew, blur→sharp.
- **Exit** — як зникає: дзеркально входу чи інакше; чи затримується.
- **Тривалість** — мс. <300мс=миттєво/чітко; 400-800мс=плавно/преміально; >1000мс=кінематографічно/повільно.
- **Easing** — крива: ease-out (швидкий старт→м'яке гальмо, найприродніше), expo/quart (драматичне гальмо), spring/back (пружність, грайливість, «перельот»), linear (механічно, рідко), cubic-bezier кастомні.
- **Stagger** — затримка між сусідніми елементами (мс). Створює «хвилю»/каскад. 30-80мс=щільна хвиля; 100-200мс=виразний каскад.
- **Зв'язність (orchestration)** — чи рух одного тягне інші; чи все підв'язано до спільного драйвера (scroll progress / курсор / таймлайн); parent-child залежності.
- **Тригер** — scroll-into-view, scroll-progress (scrub), hover, load, click.

### Емоційний словник руху (загальні правила, поповнюється):
- Швидко + ease-out + малий stagger → чіткість, впевненість, «продукт працює» (tech).
- Повільно + expo + великий stagger → розкіш, споглядання, вага кожного елемента (luxury).
- Spring/back + bounce → грайливість, дружність, людяність (creative/consumer).
- Clip-path reveal (шторка) → відчуття «відкриття завіси», театральність, передчуття.
- Blur→sharp на вході → фокусування уваги, «проявлення з туману», м'якість.
- Scrub (рух прив'язаний до scroll-позиції, не до часу) → відчуття контролю, «я керую», прямий зв'язок жесту й результату.

*(поповнюється з кожним сайтом — конкретні заміри)*

### Виміряні рецепти (реальні цифри з сайтів):
- **Tendril — «театральне розкриття»:** `clip-path + opacity` разом, **1s**, `cubic-bezier(0.65,0,0.35,1)` (easeInOutCubic, симетрична S). Ефект: завіса піднімається, вагоме преміальне розкриття. Триярусний темп: 1s контент / 0.4-0.6s UI / 0.2s мікро-відгук.
- **14islands — «каскадна хвиля» (прогресивний stagger):** базова поява `opacity+transform` @0.3s, АЛЕ ланцюг transition-delay із кроком **+0.15s** (0.3→0.45→0.6→...→1.8s) + наростаюча тривалість. Одна easing на весь сайт (`cubic-bezier(0.25,0.46,0.45,0.94)` easeOutQuad). Ефект: оркестроване розгортання, сцена оживає елемент за елементом, ведення ока. Стек: Next+React+WebGL+Lenis (наш цільовий!).

### STAGGER-ПРАВИЛА (як елементи пов'язані й входять — виведено):
- **Крок затримки** між сусідами визначає характер хвилі: 0.03-0.08s = щільна швидка хвиля (майже разом); 0.15s = виразний каскад (видно течію); >0.2s = повільне врочисте розгортання.
- **Зв'язність через спільну криву:** одна easing-крива на весь сайт + варіація delay/duration = елементи відчуваються як ланки одного ланцюга, не незалежні.
- **Тригер:** scroll-into-view запускає хвилю коли секція входить у в'юпорт (а не все на load).
- **Подвійне наростання** (delay + duration разом зростають) = «розтягнута хвиля», максимально плавне розгортання.
- **Fade-ієрархія (коли є WebGL-фон):** якщо 3D-сцена — головна зірка (IG), то DOM-елементи рухаються делікатно через чисті opacity-fade (не різкі transform/clip), щоб не конкурувати зі сценою. opacity 182× проти transform одиниць. → ієрархія уваги: 3D веде, DOM делікатно проявляється.

### ШКОЛИ РУХУ (темп культурно/нішево обумовлений):
- **Європейсько-американська** (Cuberto/Dogstudio/Tendril/Zera): повільно **1-1.2s**, драматично, рух = подія/спектакль. Криві: expo/quint/cubic.
- **Японська** (Garden Eight): швидко **0.3-0.6s**, стримано, рух = служба контенту (wabi-sabi: стриманість як вишуканість). Криві: м'який easeOut.
- **Performance/інженерна** (basement): ультрашвидко **0.06-0.3s**, snappy, рух = заява про швидкість. Криві: Material `cubic-bezier(0.4,0,0.2,1)` + linear для мікро.
→ Для нас: ОБИРАТИ темп свідомо під бажану емоцію. Повільно=вага/розкіш/спектакль; швидко=точність/легкість/чуйність; ультрашвидко=performance/реактивність.

### КІЛЬКІСТЬ РУХУ = вибір під нішу (не лише швидкість) ★ (виміряно):
Окрім ШВИДКОСТІ, ніша диктує саму КІЛЬКІСТЬ руху:
- **Спокійні ніші** (архітектура/luxury-real-estate): майже НЕМАЄ руху — Telha Clarke = ~6 переходів на всю сторінку, 1 крива easeOutQuint, color+translate. Контент-зірка, рух невидимий.
- **Tech/motion/gaming**: багато руху — 50-90 переходів, складна оркестрація, WebGL-сцени.
→ Правило: кількість руху ОБЕРНЕНО пропорційна «спокою»/преміальності-через-тишу ніші. Перед побудовою питати: ця ніша хоче спектакль чи тишу?

### РУХ ЯК ДЕКЛАРАЦІЯ ЦІННОСТЕЙ (мета-патерн) ★:
Темп/характер руху «говорить» позиціонування студії ЩЕ ДО тексту:
- Tendril 1s вагомо → «дороге кіно».
- Cuberto 1.2s expo → «елегантні продукти».
- basement 0.06-0.3s Material+linear → «швидке, що працює».
- Garden Eight 0.3-0.6s стримано → «японська точність».
→ При побудові НАШОГО сайту: спершу вирішити, яку цінність має «сказати» рух, тоді обрати темп+криву. Рух = стратегія, не оздоблення.

## A. ЯК МОУШН ВИКЛИКАЄ ЕМОЦІЮ (механіки → емоція)
Формат: прийом → як працює → яку емоцію цілить → де бачили.

### A1. Scroll-driven камера крізь 3D-сцену (scroll = час)
- Механіка: скрол керує позицією камери в 3D-просторі, не прокруткою DOM.
- Емоція: відчуття подорожі/занурення, «я рухаюсь крізь світ», контроль + відкриття.
- Бачили: Lusion (tunnels→astronaut), Zera flythrough (p9).

### A2. Matcap-освітлення + емісійні кольори на ч/б базі
- Механіка: дешеве «світіння» матеріалу + чисті насичені RGB на чорному/білому.
- Емоція: цифрова магія, «дороге й технологічне», hi-tech wonder.
- Бачили: Lusion (electric blue/lime/magenta на ч/б).

### A3. Важкий прелоадер як ритуал входу
- Механіка: тривале завантаження з анімацією перед розкриттям сцени.
- Емоція: передчуття, «зараз буде щось велике», знижує буденність.
- Бачили: Active Theory (Hydra-прелоадер), Resn.
- ⚠️ Ризик: на конверсійних нішах вбиває (тому /webdeveloper Zera — без прелоадера).

### A4. SDF-текст у WebGL (заголовки в 3D, не DOM)
- Механіка: типографіка рендериться в шейдері → може спотворюватись, плавати, реагувати.
- Емоція: текст «живий», частина світу, а не накладений зверху.
- Бачили: Immersive Garden (data-webgl-title).

### A5. «Stop-scroll» патерн-розрив
- Механіка: візуал/рух, що ламає звичку автоскролу (несподіваний 3D, рух проти очікування).
- Емоція: мимовільна зупинка уваги, цікавість, «що це?».
- Бачили: Noomo (кредо «make people stop scrolling» ×3).

### A6. Glassy 3D tiles / golden-hour aesthetics
- Механіка: скляні напівпрозорі 3D-плитки; тепле золоте світло сцени.
- Емоція: преміальність/розкіш (glass = дорого, hi-tech); тепло/ностальгія (golden hour).
- Бачили: Noomo (Salesforce glassy tiles; Coinbase×Warriors golden-hour).

## B. ЯК ПЕРЕДАЮТЬСЯ СЕНСИ (копірайт + наратив + послідовність)
### B1. Маніфест-заголовок «трансценденції»
- Прийом: H1 — обіцянка виходу за межі звичного, абстрактно-піднесено.
- Приклад: IG «Transcend anything seen or felt before...»; Zera «BUILT TO BE FELT. NOT JUST SEEN.»
- Сенс: продаємо ВІДЧУТТЯ/трансформацію, не послугу.

### B2. Якірний контраст «не X, а Y»
- Прийом: визначення через заперечення.
- Приклад: Zera «The difference isn't tools, it's execution»; ScrollCraft «not the AI, the craft».
- Сенс: перевизначає критерій оцінки на свою користь.

### B3. Тріади команд-дієслів
- Прийом: три коротких ударних дієслова поспіль.
- Приклад: Detail Driven «Strip. Cleanse. Reveal.»
- Сенс: ритм + відчуття процесу/майстерності.

### B4. Позиціонування через власний інструмент
- Прийом: студія наголошує на ВЛАСНОМУ движку/методі як перевазі.
- Приклад: Active Theory «our industry-leading web toolset».
- Сенс: «ми не аутсорс, ми винахідники» → виправдовує високу ціну.

### B5. Вимірювана емоційна ціль замість абстракції
- Прийом: обіцянка конкретного емоційно-поведінкового результату.
- Приклад: Noomo «make people stop scrolling» (×3); «ignite emotions and provoke action».
- Сенс: продаємо результат-відчуття, який клієнт може уявити.

### B6. «Story dictates the medium» (медіум під історію)
- Прийом: декларація, що засіб (WebGL/відео/editorial) обирається під наратив, не навпаки.
- Приклад: Noomo.
- Сенс: захист від «3D заради 3D»; гнучкість = зрілість студії.

### B7. Олюднення high-tech
- Прийом: поряд з технологією — слова про «human/accessible/emotional».
- Приклад: Noomo «INNOVATE – WITH A HUMAN TOUCH», «accessible human experiences».
- Сенс: контрбаланс холодному 3D; знижує бар'єр для enterprise-клієнта.

### B8. Фокус на клієнті, не на собі
- Прийом: явна заява «we're here to talk about YOU, not ourselves».
- Приклад: Noomo.
- Сенс: інверсія самопрезентації → довіра.

## C. СТИЛЬ ПІД НІШУ (який підхід де працює)
- **Luxury / fashion / watchmaking** → антиква-display + галерейна нейтральна база + дозований WebGL + повільний темп. (Immersive Garden, Bonhomme; пор. Telha-архітектура.)
- **Tech / 3D-демо / innovation** → нейтральний гротеск + чорна/біла база + насичений WebGL + швидкий точний моушн. (Lusion, Active Theory, Resn.)
- **Конверсійні (sales/курси/бізнес)** → БЕЗ важкого WebGL, швидкість, текст продає. (Zera /webdeveloper.)
- **Архітектура / спокійні** → стриманість: 1 шрифт, монохром, повільний темп, без WebGL. (Telha Clarke.)
- **Авто/детейлінг/преміум-послуга** → gold-on-black + Process-стадії + kinetic-акценти. (Detail Driven.)

- **Enterprise / B2B-tech storytelling** → нейтральний/геометричний гротеск (Neue Machina) + м'яка база (périwinkle) + 3D як ПОЯСНЕННЯ складного продукту (glassy tiles, spatial metaphors) + прозорий бюджет-фільтр. (Noomo: Salesforce/AMD/Intel/Coinbase.) Мета: зробити складне зрозумілим і емоційним для CIO/девелоперів.
- **Web/brand/editorial студія** → ПОВТОРЮВАНИЙ ПАТЕРН (Exo Ape/Antinomy/Tomorrow Bureau/Bürocratik/Obys): дисципліна ОДНОГО гротеску (часто власного) + біла АБО прозора база + рух у GSAP (мінімум CSS-transitions) + WebGL відсутній або точкові canvas-острівці + великий фото/відео-контент домінує над ефектами + портфоліо з тег-системою послуг. Емоція: впевнений спокій, «контент і типографіка — зірки, не ефекти». Дисципліна одного шрифту = 5+ підтверджень (Antinomy/Bürocratik/ExoApe/TomorrowBureau + Telha з web).

- **Brand + Motion гібрид** → антиква-display × модний гротеск (Saol Display × Neue Montreal — Unseen). Антиква з'являється в motion-ніші САМЕ коли студія робить ще й брендинг/editorial, не чистий 3D-tech. Сигнал «ми про сенс/бренд», не лише про технологію.

## D. ДИХОТОМІЯ СТРАТЕГІЙ (наскрізна рамка)
«СПЕКТАКЛЬ» (WebGL/3D/emotion, emotional ніші) ↔ «СТРИМАНІСТЬ» (типографіка/монохром/повільність, спокійні ніші).
Вибір диктує НІША й задача сторінки, не рівень майстерності. (3+ підтвердження.)

### МЕТА-ВИСНОВОК: СТЕК НЕ ВИЗНАЧАЄ РІВЕНЬ ★★★ (найважливіше)
Спектр стеків серед AWARD-студій — від нуля бібліотек до власного движка:
- **Gretel** (Nike/Netflix/MoMA) — на **Squarespace** (no-code!).
- **Order** (Herman Miller/Kickstarter) — vanilla, БЕЗ WebGL/GSAP/фреймворку, 11 шрифтів.
- **Cuberto / Use All Five** — award-моушн БЕЗ WebGL (чистий GSAP+CSS).
- **Resn** — Three.js r84 + GSAP2 (стек 2017, «застиглий»).
- **14islands / North Kingdom / Robin Noguier** — Next + React + WebGL + Lenis (сучасний).
- **Immersive Garden / Noomo / Garden Eight / Antinomy** — Nuxt + WebGL2.
- **Lusion / Active Theory** — власний движок (vanilla WebGL2 / Hydra).
**Висновок:** рівень дає КОНЦЕПЦІЯ + ВИКОНАННЯ + ТИПОГРАФІКА + КОНТЕНТ, не технологія. Стек обирається під ПОЗИЦІОНУВАННЯ:
- Сайт=демо можливостей (студія продає інтерактив) → потужний WebGL (Lusion).
- Сайт=вітрина робіт (студія продає брендинг/репутацію) → простий стек ОК (Gretel/Order).
**Для НАС (будуємо ім'я з нуля):** ми ще не маємо репутації Gretel → наш сайт МУСИТЬ бути демо-атракціоном (доводить уміння). Але клієнтські сайти — стек під задачу клієнта, не під наше его. Звільняємось від технологічного снобізму, але памʼятаємо про етап.
