# D_aqryl — aqryl.app (РОЗБІР, TIER 1-2 fintech/SaaS dashboard) ✅
> Awwwards. Трейдинг-журнал + аналітика (fintech SaaS). Next.js + нативна плавність. Dashboard-показ (не phone).
> Цінний: fintech/dashboard-продукт показ + tech-копірайт через біль ЦА.

## 1. СТЕК: Next.js + БЕЗ Lenis/GSAP (нативно). 15 екранів. Geist + Geist Mono (Vercel-стек, tech-дефолт) + Helvetica заголовки. Світла база.

## 2. ТОКЕНИ: Helvetica/Geist 36px H1 weight 400 ls-0.9 (стримано) + Geist Mono (дані/цифри). Tech-нейтральна типографіка (продукт-фокус, не крик).

## 3-4. СТРУКТУРА (fintech/dashboard SaaS — показ робочого простору):
1. HERO — «Your trades deserve more than guessing» (біль: хаос трейдингу).
2. «Everything you need, all in one workspace» — огляд продукту (dashboard-візуал).
3. ФІЧІ через сценарій-вигоду: «Journal that still reads clean months later», «Calendar built around sessions not noise», «Analytics month to day», «Risk tooling that stays visible not loud».
4. SHARING — публічна сторінка результатів, share trade.
5. АНАЛІТИКА — метрики (win rate, entry streak) як візуали.
6. CTA — Download.
- 10 dashboard-візуалів (скріни робочих екранів продукту).

## 6-7. КОПІРАЙТ (fintech/SaaS — майстер-клас «фіча через сценарій»):
- Big Idea через біль: «Your trades deserve more than guessing» (емоція трейдера: невпевненість).
- ФІЧІ НЕ як функції, а як СЦЕНАРІЇ-вигоди: «Journal that still reads clean months later» (не «журнал угод» — а ЯК він допоможе через місяці). «Calendar built around sessions, not noise» (контраст з конкурентами).
- Антитези-контрасти: «visible, not loud», «sessions, not noise» (виділення підходу).
- Tech-точність: Geist Mono для цифр/метрик (win rate, streak).
- Тон: спокійний-професійний, для серйозних трейдерів. Без хайпу.

## 8. ЕМОЦІЯ: контроль, ясність, професійність. Анти-хаос (порядок у трейдингу).

## 9. ВІДТВОРЮВАНІСТЬ 🟢: Next + нативна плавність. Dashboard-візуали = скріни/відео продукту. Geist безкоштовний.

## 10. ЗБАГАЧУЄ P_saas_landing (fintech/dashboard варіант):
- DASHBOARD-показ (не phone): великі скріни робочого простору продукту, кожна фіча = свій екран.
- ФІЧА ЧЕРЕЗ СЦЕНАРІЙ-вигоду, не назву функції («reads clean months later», не «журнал»).
- Антитеза-контраст з конкурентами («sessions, not noise», «visible, not loud»).
- Geist Mono для метрик/цифр (fintech-точність).
- Big Idea через біль ЦА (невпевненість/хаос).
