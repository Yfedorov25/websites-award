## ЧАСТИНА 2. ВЕРДИКТ ПО КОЖНОМУ — чи повторимо 1:1
═══════════════════════════════════════════════════════

### 🔴 LUSION — НЕ повторимо 1:1 (і не треба)
- **Чому ні:** власний WebGL-движок (не Three/R3F), virtual scroll повністю перехоплений у шейдери, per-letter 3D-reveal у власному рендері, multi-canvas композитинг. Це роки R&D команди топ-світу.
- **Що З НЬОГО беремо:** токен-дисципліну (12-кол сітка 2vw, radius 20px, clamp padding), RGB-акценти на ч/б, kinetic marquee тег-стрічок (🟢), per-letter reveal через SplitText (🟡 наближено).
- **Чесно:** «Lusion-lite» = 30-40% враження. Клієнт із бюджетом $3-5к цього не отримає.

### 🔴 ACTIVE THEORY — НЕ повторимо 1:1 (найскладніший)
- **Чому ні:** движок Hydra з 8 Web Workers, увесь контент (навіть текст) у WebGL, compiled-шейдери, Draco+Basis. Найрадикальніший scene-as-site.
- **Що беремо:** Draco+KTX2 оптимізацію (🟡 через R3F drei — це ГОЛОВНИЙ трофей), headless CMS (🟢), аудіо-іммерсію Howler (🟢).
- **Чесно:** 1:1 недосяжно. Але переймаємо те, що робить наш 3D плавним.

### 🟡 ERA — повторимо аналог на ~80% (наш RE-шаблон)
- **Чому 🟡:** дозований 3D (5 острівців, не весь сайт) + кастомний reveal-движок. Найдосяжніший з «важких».
- **Що повторимо 1:1 (🟢):** 193-токенну систему (особливо 99 кольорів з ролями), монументальну типографіку (171px/lh0.93/негативний трекінг), 3s reveal (GSAP), копірайт-підхід (епоха/місце/імена).
- **Що 🟡:** 3D-карта району (R10 на R3F), sticky mask-reveal (R11), дозовані 3D-моделі.
- **Альтернатива 🟢:** замість real-time 3D-моделей → pre-rendered обертання (відео). Дає 90% враження ERA.
- **Чесно:** це РЕАЛЬНО повторюваний еталон для нас. 80-90% враження досяжно за 2-3 тижні.

### 🟡 IMMERSIVE GARDEN — повторимо аналог на ~75% (наш emotional-шаблон)
- **Чому 🟡:** Nuxt+Three (фреймворк!), persistent WebGL-фон + scene swapping, Draco+KTX2. Не власний движок = досяжно.
- **Що повторимо (🟡):** persistent Canvas на R3F, Draco .glb + KTX2 текстури через drei, антиква-естетика, двополюсний рух 1.9s/0.25s (🟢 GSAP), Howler-аудіо (🟢).
- **Складність:** persistent WebGL з морфінгом сцен + оптимізація ассетів вимагає скіла, але це наш стек.
- **Чесно:** 70-80% враження досяжно за 3-4 тижні з добрим 3D-скілом. Це наша «стеля».

### 🟢 CUBERTO — повторимо 1:1 (~95%!)
- **Чому 🟢:** БЕЗ WebGL. Чистий GSAP+Lenis+CSS. Кастом-курсор — їхня бібліотека ВІДКРИТА (cuberto/mouse-follower на GitHub).
- **Що повторимо 1:1:** ВСЕ. fade-up @1.2s expo, розшарований stagger, кастом-курсор, морфінг border-radius, типографіка lh=1.0.
- **Чесно:** це НАЙВІДТВОРЮВАНІШИЙ award-сайт. 95% за дні. Award-рівень БЕЗ 3D.

### 🟡→🟢 14ISLANDS — повторимо на ~85% (наш базовий стек)
- **Чому:** Next+React+R3F+Lenis = буквально наш план. Home=відео (легко!), 3D точково в кейсах.
- **Що повторимо 1:1 (🟢):** easing-бібліотеку (16 кривих — копіюємо як є!), прогресивний stagger, тематичну систему кольорів, тег-портфоліо, home=відео.
- **Що 🟡:** R3F-сцени в кейсах (коли потрібні).
- **Чесно:** 85% за 2-3 тижні. Home взагалі легкий (відео).

### 🟡→🟢 NORTH KINGDOM — повторимо на ~85% (наш bold-стек)
- **Те саме що 14islands:** Next+R3F+Lenis+leva, home=showreel.
- **Що повторимо 1:1 (🟢):** easing-бібліотеку, alpha-систему кольорів (глибина темних дизайнів), FK-стиль типографіки, складену оркестрацію переходів.
- **Чесно:** 85%, home легкий (відео-showreel).

## ЧАСТИНА 3. ГОЛОВНІ ВИСНОВКИ
═══════════════════════════════════════════════════════

### Що ДІЛИТЬ сайти на «повторимо / ні»:
1. **Власний движок = 🔴** (Lusion, Active Theory). Three/R3F/GSAP = 🟡-🟢 (решта). Головний вододіл.
2. **Real-time 3D всього сайту = важко.** Дозований 3D (ERA) або 3D-в-кейсах (14islands/NK) або відео = досяжно.
3. **WebGL для тексту/всього = 🔴.** WebGL для фону/острівців = 🟡. Без WebGL (Cuberto) = 🟢.

### ЩО МИ ТОЧНО МОЖЕМО (🟢 — фундамент усіх наших сайтів):
- Уся анатомія руху GSAP: fade-up, stagger, expo/quad easing, морфінг, кастом-курсор.
- Easing-бібліотека як токени (від 14islands/NK — копіюємо 16 кривих).
- Дизайн-токени: палітри з ролями (99 як ERA), alpha-шкали (NK), сітки, clamp-типографіка.
- Lenis smooth-scroll. Монументальна типографіка. Kinetic marquee/type.
- Home=відео (Higgsfield/Kling генерує) замість real-time 3D.
- Sticky scroll-reveal, mask-reveal (clip-path), parallax.

### ЩО МИ МОЖЕМО З ЗУСИЛЛЯМ (🟡 — потрібен 3D-скіл):
- R3F-сцени точково (3D-карта, обертання продукту, 1 фокусна сцена).
- Persistent WebGL-фон (як IG) — складніше, але наш стек.
- Draco+KTX2 оптимізація ассетів (drei) — ключ до плавного 3D.

### ЧОГО МИ НЕ МОЖЕМО (🔴 — не беремось, не обіцяємо клієнту):
- Власний multi-threaded движок (Hydra), весь сайт у real-time WebGL.
- Per-letter 3D-reveal у власному рендері, складні GPGPU-симуляції на весь екран.
- → Замість цього: pre-rendered відео (80% враження, частка вартості).
