# 🔧 АНАЛІЗАТОР A3 — Збирач суперТЗ для Claude Code
> Як з діагностики (A2) + критеріїв (A1) + бібліотеки рецептів зібрати суперТЗ, по якому Claude Code будує award-результат.

## ВХІД: результат A2 (тип/тір/ніша) + сенси-копірайт (з meaning-copy-system) + обрані рецепти.

## СТРУКТУРА суперТЗ (що генерувати):
### 0. КОНТЕКСТ ПРОЄКТУ
- Бренд, Big Idea, емоція-ціль, ЦА, тон/архетип. Тір. Стек.
### 1. ГЛОБАЛЬНІ ТОКЕНИ (з A1 + ніша-профіль)
- Типографіка: display+body шрифти (конкретні), повна шкала clamp по ролях, lh/ls (за критеріями 1-6).
- Палітра: 6+ токенів з ролями + alpha-шкала (критерії 7-11).
- Сітка: 12-кол, гаттери, max-width, бічні clamp, 8pt-ритм (12-16).
- Easing-бібліотека: 6+ `--ease-*` (18). Lenis-конфіг (22).
### 2. ПОСЕКЦІЙНЕ ТЗ (для кожної секції з обраного рецепта)
Для КОЖНОЇ: призначення + копірайт(фінальний, за анти-слоп) + структура DOM + типографіка + анімація(код GSAP) + рецепт-ref + ассети + мобільний + критерії-чек.
### 3. ГЛОБАЛЬНИЙ РУХ
- Прелоадер (31), scroll-reveal система (21), переходи секцій (24), hover-патерни (26-30), reduced-motion (25).
### 4. ПЕРФОРМАНС-БЮДЖЕТ
- LCP<2.5/INP<200/CLS<0.1 (45), ассети (47), home-легкість (46), мобільний (48), SEO (49).
### 5. ІНСТРУКЦІЯ CLAUDE CODE
- Стек + плагіни (gstack/gsap/uiux-pro/design-engineer/council-llm).
- Послідовність білду (токени→глобальне→секції→рух→QA).
- ВЕСЬ копірайт через анти-слоп перед вставкою.
- Самоперевірка по 50 критеріях A1 (45+/50 = award).

## ПРАВИЛО ЯКОСТІ:
ТЗ готове, коли: кожна секція має конкретні ЧИСЛА (не «гарний відступ» а «clamp(80px,12vh,160px)»), кожна анімація має код, копірайт чистий, пройдено 45+/50 критеріїв. Якщо десь «на розсуд Claude Code» — це діра, доробити число.
